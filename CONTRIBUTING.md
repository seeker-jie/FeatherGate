# Contributing Guidelines

Thank you for your interest in FeatherGate! We welcome contributions in all forms.

## How to Contribute

## Language Preferences

This project primarily uses **English** for:
- README.md (main documentation)
- Commit messages  
- Code comments
- Issue reports (either English or Chinese acceptable)

Chinese documentation is available in:
- README_ZH.md (comprehensive Chinese version)

## How to Contribute

### Reporting Issues

When reporting issues in GitHub Issues, please include:
- Problem description
- Steps to reproduce
- Expected behavior
- Actual behavior
- Environment information (OS, Rust version, etc.)

### Submitting Code

1. Fork this repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Create a Pull Request

### Code Standards

- Run `cargo fmt` to format code
- Run `cargo clippy` to check code quality
- Ensure all tests pass (`cargo test`)
- Add tests for new features

### Language and Documentation

**Primary Language**: English (README.md, commit messages, code comments)
**Secondary Language**: Chinese documentation available in README_ZH.md
**Commit Messages**: Please use English for consistency
**Issue Reports**: Either English or Chinese is acceptable
